다음 순서로 변경사항 스테이징부터 커밋 메시지 생성 및 커밋 실행까지 처리해.

1. `git status` 로 변경된 파일 목록 확인
2. `git diff` 로 변경사항 내용 파악
3. 변경사항 분석 후 아래 컨벤션에 맞는 커밋 메시지 제안
4. 사용자 확인 후 `git add -A` 및 `git commit -m "..."` 실행

커밋 메시지 컨벤션:
- 기능 단위로 쪼개어 커밋 분할 작성
- 아래 접두어 리스트만 커밋 내용에 맞게 사용할 것
- 아래 커밋 형식 참고하여 작성
  - 제목과 본문은 빈 행으로 구분
  - 제목은 명령조로 작성
  - 본문은 72자마다 줄 바꿈

접두어 리스트:
- feat: 기능 개발 커밋
- fix: 버그 수정
- refactor: 기존 코드 리팩토링 커밋
- docs: 문서 작성 커밋
- chore: 빌드/설정 변경
- test: 테스트 코드

커밋 형식:
[접두어]: [변경 내용 요약]

[본문] (필수 아님)


주의:
- 변경사항이 없으면 사용자에게 알리고 중단
- 변경사항 파악이 모호하면 사용자에게 알림
- `git add -A` 와 커밋은 반드시 사용자 확인 후 실행
- untracked 파일 포함 여부를 사용자에게 먼저 확인
